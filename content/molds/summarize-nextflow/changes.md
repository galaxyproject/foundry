# summarize-nextflow — changes

Revision history for the `summarize-nextflow` Mold. Maintained alongside `index.md` but never packaged into casts.

- **rev 11 (2026-07-20)** — conda-spec parsing fidelity: a partially-resolved `conda` directive is now either fully read or flagged. `==` pins are read as versions (`findutils==4.6.0` was recording `unknown`); inexact constraints (`>=1.17`) are preserved verbatim in the new `Tool.version_constraint` instead of being dropped; the `unknown` sentinel is filtered out of `Tool.versions[]` (nine tools across four pipelines cleaned, and `coreutils` in createtaxdb/proteinfamilies resolved to a real `["9.4", "9.5"]` divergence the sentinel had masked); the pre-`environment.yml` ternary form `conda (params.enable_conda ? "<spec>" : null)` is captured rather than being invisible; and an unreadable spec inside an otherwise-resolving directive now warns instead of vanishing. The null `processes[].tool` FK on genuinely multi-package processes is confirmed as the intended behavior and pinned by a test. Corpus warning count unchanged at 65 across 26 pipelines; no container, `bioconda`, or FK field changed. Resolves galaxyproject/foundry#356. Schema bumped to rev 10 in lockstep.
- **rev 10 (2026-07-20)** — reference-data extraction fidelity plus remote/pinned source acquisition. Legacy literal-string `conda` directives now populate `tools[]` (nf-core/eager 43 -> 50 tools) and unresolvable directives are flagged in `warnings[]`. `reference_assets[]` drops execution/registry params and kinds multi-reference sheets; `used_by` attributes params consumed via channel construction; negative-guard rebuild detection covers the fused `ch_x = BUILDER(args).chan` form (eager 0 -> 5 rules). `Tool.versions[]` records divergent per-module version pins. Source acquisition gained remote git URLs and `--pin <tag|branch|sha>` via a detached temporary checkout that never mutates a caller-supplied tree; failures use the documented exit-code contract instead of 64. Resolves galaxyproject/foundry#349 and galaxyproject/foundry#351. Schema bumped to rev 9 in lockstep.
- **rev 9 (2026-05-06)** — top-level `Param` entries gained `format`, `hidden`, `mimetype`, `schema_group`, and `fa_icon` from `nextflow_schema.json`. Downstream Molds ([[nextflow-summary-to-galaxy-interface]], [[nextflow-summary-to-cwl-interface]], [[nextflow-summary-to-galaxy-data-flow]]) can consume nf-schema metadata structurally instead of re-parsing the source schema. Resolves galaxyproject/foundry#186; closes the open question at the foot of [[nextflow-params-to-galaxy-inputs]]. Schema bumped to rev 7 in lockstep. Real-fixture integration assertions added against `nf-core/demo` and `nf-core/rnaseq`.
- **rev 8 (2026-05-05)** — sample-sheet schemas promoted to a first-class top-level `sample_sheets[]` field with structured `SampleSheet` and `SampleSheetColumn` shapes. New §3.5 procedure step covers resolution from `nf-schema`, `samplesheetToList`, `splitCsv`, and `ad-hoc` discovery branches. Resolves the open question in [[nextflow-workflow-io-semantics]] and galaxyproject/foundry#177; downstream variant selection lives in [[nextflow-summary-to-galaxy-interface]] / [[nextflow-summary-to-galaxy-data-flow]] backed by [[galaxy-sample-sheet-collections]]. Schema bumped to rev 6 in lockstep.
- **rev 7 (2026-05-02)** — CLI package now sweeps `include { X as Y } from ...` statements across workflow and subworkflow files to populate `processes[].aliases`, tested against bacass repeated imports (`MINIMAP2_ALIGN`, `FASTQC`).
- **rev 6 (2026-05-02)** — CLI package now parses multi-dependency module `environment.yml` files into separate Bioconda-backed `tools[]` entries, tested against bacass modules such as `minimap2/align` and `samtools/sort`.
- **rev 5 (2026-05-02)** — CLI package hardened against `nf-core/bacass` profile config expressions: resolves `params.pipelines_testdata_base_path + '...'`, fetches samplesheet-referenced remote files, hashes them, and optionally localizes them under `--test-data-dir` while preserving original URLs.
- **rev 4 (2026-05-01)** — second cast against `nf-core/bacass @ 2.5.0` (33 processes, 9 nf-test files, 11 test profiles) exposed two patterns the first cast couldn't see: process aliasing via `include { X as Y }` (six distinct alias-rename patterns in bacass) and the per-test-file structure of nf-test fixtures. §4 grew the alias-sweep rule; §7 split into `test_fixtures` + `nf_tests[]` with structured snapshot extraction.
